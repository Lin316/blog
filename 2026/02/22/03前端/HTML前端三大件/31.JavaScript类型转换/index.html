JS 类型转换概述

JavaScript 是弱类型语言，不同类型之间可以自动或手动转换。

常见类型转换分为：

  • 隐式类型转换（自动转换）
  • 显式类型转换（手动转换）

JS 在以下场景会自动进行类型转换：

  • 运算符运算
  • 条件判断
  • 比较操作
  • 字符串拼接

最佳实践建议：

  • 优先使用 ===，避免 == 带来的隐式类型转换陷阱
  • 尽量使用显式类型转换（Number、String、Boolean 等）
  • 不要依赖复杂的隐式类型转换逻辑
  • 避免在生产代码中使用“难以读懂”的 == 比较

== 与 === 区别（核心）

=== 严格相等：不做类型转换进行比较。

== 抽象相等：会尝试进行类型转换，再比较，转换规则较复杂。

类型转换优先级简述

  • 数字运算时：尽量全部转为 Number
  • 字符串 + 运算：只要有字符串，就做拼接
  • == 比较时：
    • Boolean → Number
    • String → Number
    • Object → ToPrimitive
