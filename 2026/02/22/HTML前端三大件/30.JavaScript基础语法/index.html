通过 <script> 标签写在 HTML 文件内部。

特点：

  • JS 写在 HTML 内部
  • 适合少量代码、示例和测试
  • 不利于大型项目的维护

把 JS 写在单独的 .js 文件中，通过 src 引入。

index.html：

特点：

  • 结构清晰（HTML 负责结构，JS 负责逻辑）
  • 方便复用与维护
  • 开发中最常用的方式

<script> 标签的加载方式

<script> 是 HTML 中用于加载和执行 JavaScript 的标签。
不同的加载方式会直接影响 页面渲染速度、执行顺序、是否阻塞 DOM 解析。

默认加载方式：

  1. 浏览器解析 HTML
  2. 遇到 <script>
  3. 暂停 HTML 解析
  4. 下载 JS
  5. 执行 JS
  6. 执行完毕后继续解析 HTML

defer 延迟加载：

  1. 解析 HTML
  2. 遇到 <script defer>
  3. 不会阻塞 HTML 解析
  4. 后台下载 JS
  5. HTML 解析完成后
  6. 按顺序执行所有 defer 脚本

async 异步加载：

  1. 解析 HTML
  2. 遇到 <script async>
  3. 后台下载 JS
  4. 下载完成后立即执行
  5. 执行时会暂停 HTML 解析

加载方式对比总结：

方式 是否阻塞HTML解析 是否保证顺序 执行时机
默认 ✅ 阻塞 ✅ 保证 立即执行
defer ❌ 不阻塞 ✅ 保证 DOM 解析完成后
async ❌ 下载不阻塞 ❌ 不保证 下载完成立即执行

JS 变量概述

变量用于存储数据。

JavaScript 中有 3 种声明变量的方式：

  • var（旧写法，不推荐）
  • let
  • const

var 使用示例：

特点：

  • 可以重复声明
  • 没有块级作用域（只有函数作用域）
  • 存在变量提升
  • 容易造成全局污染

特点：

  • 不能重复声明
  • 有块级作用域
  • 不允许在声明之前使用（暂时性死区）
  • 相比 var 更安全

特点：

  • 声明时必须赋值
  • 不能重新赋值
  • 有块级作用域
  • 推荐优先使用 const，表示“不打算修改”的变量
