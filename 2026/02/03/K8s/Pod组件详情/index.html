Pod 组件 yaml 配置

Pod 组件 yaml 配置详情

容器 imagePullPolicy 配置

注：如果镜像的 tag 有具体的版本号，那么 imagePullPolicy 默认值为：IfNotPresent。

​ 如果镜像的 tag 是 latest，那么 imagePullPolicy 默认值为：always。

imagePullPolicy 取值 拉取策略
Always 只用远程仓库的镜像创建容器
IfNotPresent 如果本地仓库有镜像，就用本地仓库的镜像创建容器，如果本地仓库没有镜像，就用远程仓库的镜像创建容器。
Never 只使用本地仓库的镜像创建容器，如果本地仓库没有镜像，就报错。

容器 command 配置

command 用来配置容器的启动命令，如果yaml 配置了 command，那么 Dockerfile 的 CMD 启动命令配置就不会生效，以 command 配置的启动命令为准。

容器 resources 配置

limits：用于配置容器运行时最多可以占用的资源，当容器占用的资源超过上限时会被终止，并进行重启。

requests：用于配置运行容器最少需要的资源，如果资源没有达到最低要求，容器无法启动。

cpu：配置 CPU 的核心数，可以为整数或小数。

memory：配置内存大小，单位可以选 Gi、Mi、G、M。

Pod 生命周期

Pod 生命周期：创建 Pod、运行初始化容器、运行主容器、终止 Pod。

Pod 在生命周期中会出现的5种状态：

  • 挂起（Pending）：Pod 已被创建，但还没被调度到节点运行，例如拉取容器镜像时。
  • 运行中（Running）：pod 成功被调度到节点上，并且至少有一个容器正在启动或者正在运行。
  • 成功（Succeeded）：pod 内的所有容器都正常退出，例如一次性任务成功执行结束。
  • 失败（Failed）：pod 内的所有容器都已终止，但至少有一个容器异常终止，比如逻辑报错、内存溢出等。
  • 未知（Unknown）：无法获取 pod 状态，通常由网络通信失败导致的。

初始化容器

初始化容器是一种在主容器启动前执行的特殊容器，用于完成一次性的准备工作。

初始化容器的特点：

  • 顺序执行：初始化容器需要按顺序执行，只有上一个初始容器执行成功了，才会执行下一个初始化容器。
  • 必须全部执行成功：只要有一个初始化容器执行失败，pod 会重启，主容器不会启动。
  • 只执行一次：每次创建 pod 的时候都会执行一次，执行完成后会退出，不会长期运行。

初始化容器的作用：

  • 准备运行环境：下载依赖包、解压文件、生成配置、初始化数据库脚本。
  • 注入主容器缺少的工具：例如 curl、bash 等通过共享 Volume 提供给主容器使用。
  • 等待依赖服务就绪：如等待下游服务启动后再启动主容器。
