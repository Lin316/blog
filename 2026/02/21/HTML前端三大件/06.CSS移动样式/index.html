CSS 渐变是一种特殊的背景图像，用于在两种或多种颜色之间创建平滑过渡效果，本质上是 background-image 的一种形式。

3D 空间转换

CSS 空间转换介绍

CSS 3D 空间转换是基于三维坐标系（X、Y、Z）对元素进行旋转、位移、缩放等操作，让页面产生“立体效果”。

在 3D 空间中：

  • X 轴：水平方向（左 → 右）
  • Y 轴：垂直方向（上 → 下）
  • Z 轴：垂直屏幕方向
    • 正值：向屏幕外（靠近用户）
    • 负值：向屏幕内（远离用户）

位移：translate3d

translateZ() 必须给父元素配置 perspective 才能明显看到立体效果。

缩放：scale3d

scale3d() 必须给父元素配置 transform-style: preserve-3d 才能明显看到立体效果。

视距：perspective

指定了观察者与 Z=0 平面的距离，为元素添加透视效果，通常加在父元素上：

  • 数值越小 → 透视效果越强（更“夸张”）
  • 数值越大 → 越接近“平面”
  • 根据经验，取值范围一般在 800px~1200px 之间

子元素空间类型 transform-style

控制子元素在 3D 空间中是否保留 3D 效果：

  • flat（默认）：子元素被压扁为 2D
  • preserve-3d：子元素保留 3D

背面可见性 backface-visibility

控制元素翻转 180° 后，背面是否可见：

  • visible：背面可见（默认）
  • hidden： 背面隐藏

自适应布局 rem

rem 是一个相对于根元素（html）的 font-size 的单位。

计算公式：1 rem = html 的 font-size优点

rem 优点：

  • 全局可控
  • 不会嵌套叠加
  • 适合整体响应式设计

rem 缺点：

  • 必须控制 html font-size
  • 不适合纯按屏幕比例缩放的布局

自适应布局 vw

1vw 就是屏幕宽度的 1%：

  • 视口宽 390px → 1vw = 3.9px
  • 视口宽 750px → 1vw = 7.5px
  1. vw 的问题：比 100% 宽还大（没复现出来）

vw 包含滚动条，会导致比 100% 宽还大，建议把 width: 100vw; 改成 width: 100%;

含义：最小 16px，随 5vw 增长，但最大不超过 48px。

自适应布局 vh

1vh 就是屏幕长度的 1%：

  • 视口长 200px → 1vh = 2.0px
  • 视口长 850px → 1vh = 8.5px

vh 可能导致的问题

在一些移动浏览器上：

  • 浏览器地址栏/底部工具栏展开或收起，会改变“可视区域高度”
  • 100vh 可能包含“被工具栏占用的高度”，导致页面内容被顶住或出现跳动

现代解决方案：

  • dvh：动态视口高度（随地址栏变化，更推荐）
  • svh：小视口高度（地址栏展开时的最小高度，更保守）

用法总结

  • 想“整体缩放一套 UI”（间距、圆角、字号一致变大/变小）：优先 rem
  • 想“某个东西永远占屏幕的百分比”（比如 2 列卡片、左右留白）：优先 vw
  • 想做“首屏/全屏高度”：优先 dvh/svh（其次 vh）
  • 字号：常用 rem 或 clamp(px, vw, px)，避免纯 vw
