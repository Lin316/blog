盒子样式

页面中的每一个元素，本质上都是一个矩形盒子。

CSS 的一个标准盒子从内到外由 4 个部分组成：

盒子模型的组成

content（内容区域）

元素真正显示内容的区域。

  • width：内容宽度
  • height：内容高度

说明：默认情况下，width / height 只控制 content 区域。

padding（内边距）

内容和边框之间的距离。

简写规则：

  • padding: 上 右 下 左;
  • padding: 10px 20px; 表示 上下 10px，左右 20px
  • padding: 10px; 四个方向都是 10px

特点：

  • 会撑大盒子
  • 背景颜色会覆盖 padding 区域

border（边框）

围绕 padding 的边线。

等价写法：

常见 border-style：

  • solid：实线边框。
  • dashed：虚线边框。
  • dotted：点状边框，比 dashed 更细碎。
  • double：双线边框。

margin（外边距）

控制盒子与其他盒子的距离。

特点：

  • 不影响背景颜色，始终是透明区域
  • 会产生“外边距合并”问题

盒子模型

标准盒模型（content-box）

计算方式：

盒子模型计算公式

在 content-box 下：

总宽度 = width
+ padding-left + padding-right
+ border-left + border-right
+ margin-left + margin-right

在 border-box 下：

总宽度 = width
+ margin-left + margin-right

开发中常遇到的问题

为什么设置 width: 100% 还会换行？

原因：

  • 100% + padding + border > 父元素宽度

解决：

为什么两个 div 之间有“多余空隙”？

可能原因：

  • margin 合并
  • inline-block 元素之间的空白字符
  • 浏览器默认 body 的 margin

Flex 布局

Flex 布局是现代 CSS 中用于实现弹性、响应式布局的核心方案，相比传统的 float + position 更简单、可控。

Flex 布局的基本概念

任何一个容器都可以指定为 Flex 布局：

行内元素也可以使用 Flex 布局：

设为 Flex 布局后：

  • 该元素称为“容器”（flex container）
  • 它的所有直接子元素称为“项目”（flex item）
  • 项目默认沿主轴（main axis）排列，另一条轴为交叉轴（cross axis）

容器上的常用属性

flex-direction 主轴方向

控制主轴方向（项目的排列方向）。

常见取值：

  • row：主轴为水平方向，起点在左侧
  • row-reverse：水平方向，起点在右侧
  • column：垂直方向，起点在上方
  • column-reverse：垂直方向，起点在下方

flex-wrap 是否换行

默认所有项目都在一行内，flex-wrap 控制是否换行。

常见取值：

  • nowrap：不换行（默认）
  • wrap：换行，第一行在上方
  • wrap-reverse：换行，第一行在下方

flex-flow 复合写法

flex-flow 是 flex-direction 和 flex-wrap 的简写。

语法：

  • flex-flow: <flex-direction> <flex-wrap>;

justify-content 主轴对齐方式

控制项目在主轴上的对齐方式（假设主轴为从左到右）。

常见取值：

  • flex-start：靠主轴起点对齐（默认）
  • flex-end：靠主轴终点对齐
  • center：居中对齐
  • space-between：两端对齐，项目间距相等
  • space-around：项目两侧间距相等，项目间距是两端间距的两倍

align-items 交叉轴对齐方式

控制项目在交叉轴上的对齐方式（假设交叉轴从上到下）。

常见取值：

  • flex-start：交叉轴起点对齐
  • flex-end：交叉轴终点对齐
  • center：交叉轴中点对齐
  • baseline：项目的第一行文字的基线对齐
  • stretch：如果项目未设置高度或设为auto，将占满整个容器的高度（默认）

align-content 多行交叉轴对齐

当存在多行（flex-wrap 生效）时，控制多行整体在交叉轴上的对齐方式；只有一行时无效。

常见取值：

  • flex-start：交叉轴起点对齐
  • flex-end：交叉轴终点对齐
  • center：交叉轴中点对齐
  • space-between：两端对齐，行间距平均
  • space-around：项目两侧间距相等，项目间距是两端间距的两倍
  • stretch：行占满整个交叉轴

项目上的常用属性

order 排序

控制项目在主轴中的排列顺序，数值越小越靠前。

默认值为 0。

flex-grow 放大比例

控制项目在主轴剩余空间中的放大比例。

说明：

  • 默认值为 0，表示即使有剩余空间也不放大
  • 所有项目 flex-grow: 1 时，按比例平分剩余空间
  • 某个项目为 2，其他为 1 时，该项目占用的剩余空间是其他项目的两倍

flex-shrink 缩小比例

控制主轴空间不足时项目的缩小比例。

说明：

  • 默认值为 1，表示空间不足时等比例缩小
  • 某个项目设为 0 时，在空间不足时不会被压缩
  • 只有一行时 flex-shrink 生效，当存在多行（flex-wrap 生效）时不生效

flex-basis 初始主轴尺寸

设置项目在分配多余空间前占据的主轴空间。

说明：

  • 默认值为 auto，通常等同于自身内容大小或 width/height
  • 可以设定为具体长度值，例如 300px，会覆盖 width 配置

flex 综合属性（最常用）

flex 是 flex-grow、flex-shrink、flex-basis 的简写。

常见写法：

  • flex: 1; 等价于 flex: 1 1 0;，常用于自适应等分布局
  • flex: auto; 等价于 1 1 auto
  • flex: none; 等价于 0 0 auto

align-self 单个项目对齐方式

允许单个项目覆盖容器的 align-items 设置。

常见取值与 align-items 基本一致：

  • auto：默认，继承 align-items
  • flex-start / flex-end / center / baseline / stretch

Flex 布局示例：水平居中 + 垂直居中

这个写法可以非常方便地实现子元素在容器中的“完美居中”，是 Flex 布局最常用的场景之一。
