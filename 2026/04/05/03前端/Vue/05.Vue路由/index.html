路由概述

什么是路由

Vue 中的路由：路径和组件的映射关系

Vue Router 插件

Vue Router 是 Vue.js 官方的路由管理器，让构建单页应用变得简单。

核心功能：

  • 路由映射：配置路径与组件的对应关系
  • 路由导航：通过 <router-link> 实现页面跳转
  • 路由出口：使用 <router-view> 显示匹配的组件
  • 路由参数：支持动态路由、查询参数
  • 导航守卫：路由级别的权限控制
  • 路由元信息：附加路由配置信息
  • 路由动画：过渡效果
  • 路由懒加载：按需加载组件

官方文档：https://v3.router.vuejs.org/zh/

路由模式（mode）

Vue Router 提供两种路由模式：

Hash 模式（默认）

  • URL 格式：http://example.com/#/home
  • 原理：利用 URL 的 hash（# 及其后面的内容）实现路由
  • 优点：兼容性好，无需服务器配置
  • 缺点：URL 不够美观，可能对 SEO 有影响

History 模式

  • URL 格式：http://example.com/home
  • 原理：使用 HTML5 History API（pushState、replaceState）
  • 优点：URL 更美观，符合常规 URL 格式
  • 缺点：需要服务器配置支持（所有路由都返回 index.html）

缓存组件 keep-alive

基本用法

keep-alive 用来缓存动态组件或路由组件的状态，避免重复渲染，提高性能和用户体验。

缓存路由组件的状态

  • 页面切换时保留状态（表单、滚动位置等）
  • 再切回来不会重新执行 created 、 mounted 、destroyed

缓存控制

include：只缓存指定组件

exclude：排除指定组件，其它组件都缓存

生命周期钩子

被 keep-alive 包裹的组件，会多两个生命周期

  • activated：进入这个页面的时候触发
  • deactivated：离开这个页面的时候触发
