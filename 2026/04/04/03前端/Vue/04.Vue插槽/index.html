插槽概述

什么是插槽

插槽（Slot）允许父组件向子组件传递自定义的模板内容，实现组件的灵活组合和复用。

核心价值：

  • 组件内容可定制化
  • 提高组件复用性
  • 实现组件间的内容协作

插槽的分类

Vue 中的插槽主要分为三类：

类型 描述 使用场景
默认插槽 单个未命名的插槽 组件主体内容自定义
具名插槽 多个带有 name 属性的插槽 组件多个位置内容自定义
作用域插槽 子组件向父组件传递数据的插槽 父组件根据子组件数据动态渲染

注意：作用域插槽不是插槽的一种独立分类，而是插槽的一种高级用法。

具名插槽

基本概念

当组件内有多处结构需要外部自定义时，使用默认插槽无法满足需求，此时需要使用具名插槽。

具名插槽通过 name 属性为不同的插槽命名，父组件使用 v-slot:name 或 #name 语法向指定插槽传入内容。

注意事项

  2. 具名插槽必须使用 template 标签包裹
  3. 如果没有指定插槽名，默认使用 default
  4. 可以同时使用默认插槽和具名插槽

作用域插槽

基本概念

作用域插槽允许子组件向父组件传递数据。在封装组件时，有时需要让父组件根据子组件的数据动态决定渲染内容，这时就需要使用作用域插槽。

核心机制：

  1. 子组件在 <slot> 标签上绑定属性，传递数据
  2. 父组件在接收插槽时，通过 slot-scope 或 v-slot 获取传递的数据
  3. 父组件可以使用这些数据自定义渲染逻辑
