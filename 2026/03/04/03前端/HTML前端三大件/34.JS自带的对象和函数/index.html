history 对象

history 用来 操作浏览器的历史记录。

它允许 JavaScript：

  • 前进、后退页面
  • 跳转到历史记录中的某个位置
  • 修改 URL（不刷新页面）
  • 在 SPA 单页应用 中实现前端路由

常用属性

length

返回 历史记录数量。

注意：

  • 包含当前页面
  • 无法访问具体 URL（安全限制）

常用方法

back()

返回上一页。

forward()

前进到下一页。

go()

跳转到历史记录中指定的位置。

参数 含义
-1 返回上一页
1 前进一页
-2 返回两页
0 刷新页面

pushState()

向浏览器历史记录栈中新增一条记录，同时修改 URL，但不会刷新页面

参数 说明
state 状态对象
title 页面标题（基本不用）
url 新URL

history.pushState() 和 history.replaceState() 的 第一个参数 state 用来：

给当前历史记录保存一个“状态对象”

当用户 前进 / 后退 时，这个状态对象会通过 popstate 事件返回，就能拿到之前保存的数据。

pushState() 的行为是：

1 修改URL
2 新增历史记录
3 不刷新页面

localStorage 对象

对象介绍

localStorage 对象用来在浏览器中永久保存键值对数据，是前端开发中非常常用的一种 客户端存储方式。

特点是：简单、容量比 cookie 大、不会随 HTTP 请求发送到服务器。

作用：

  • 在浏览器本地存储数据
  • 数据以 key-value 形式保存
  • 数据不会过期（除非手动删除）

localStorage 的特点：

特性 说明
存储类型 键值对
存储位置 浏览器本地
生命周期 永久（除非删除）
容量 约 5MB
是否自动发送到服务器 不会
同源限制 只能同协议 + 域名 + 端口访问

同源限制说明：

这两个 localStorage 不共享。

常用属性

length

获取存储数据数量。

常用方法

方法 作用
setItem() 保存数据
getItem() 读取数据
removeItem() 删除数据
clear() 清空所有数据
key() 获取指定索引的 key

setItem()

保存数据，但只能保存字符串。

getItem()

removeItem()

sessionStorage 对象

sessionStorage 用于在 浏览器会话（session）期间存储数据。它的用法和 localStorage 很像，但 生命周期更短。

特点：

  1. 按会话存储
    • 数据只在 当前浏览器标签页 有效。
    • 关闭标签页或窗口后数据就会被清除。
  2. 同源限制
    • 只能在 同协议 + 同域名 + 同端口 下访问。
  3. 每个标签页独立
    • 即使是同一个网站，不同标签页的 sessionStorage 也是独立的。
  4. 存储格式
    • 只能存储 字符串（string）。
    • 如果要存对象，需要使用 JSON.stringify()。
  5. 存储容量
    • 一般 5MB 左右（不同浏览器略有差异）。
方法 作用
setItem() 保存数据
getItem() 读取数据
removeItem() 删除数据
clear() 清空所有数据
key() 获取指定索引的 key
