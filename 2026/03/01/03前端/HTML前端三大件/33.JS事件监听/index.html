事件监听介绍

事件监听可以理解为：当用户执行完了某个动作，比如点击、输入、获得焦点等，然后执行代码。

监听三要素：

  • 监听谁：DOM 元素
  • 监听什么：事件类型（click / input / keydown …）
  • 发生后干什么：事件处理函数（回调函数）

事件冒泡

事件流

事件流是对事件执行过程的描述，任意事件被触发时都会经历两个阶段：【捕获阶段】和【冒泡阶段】。

捕获阶段是【从父到子】的传导过程，冒泡阶段是【从子向父】的传导过程。

什么是事件冒泡

事件冒泡：当某个元素触发事件后，这个事件会从当前元素开始，沿着 DOM 树一层一层向上级元素传播，直到 document。

事件对象

事件对象介绍

事件对象：当一个事件发生时，浏览器会自动创建一个 事件对象，并把它作为参数传给事件处理函数。

这个 event 就是事件对象。

事件对象共有属性

所有事件对象都拥有这些基础属性：

属性 说明
type 事件类型
target 触发事件的元素
currentTarget 当前绑定事件的元素
bubbles 当前事件是否会进行“冒泡传播”
cancelable 该事件是否可以被 preventDefault() 取消默认行为。
defaultPrevented 是否已阻止默认行为
eventPhase 当前处于哪个阶段
timeStamp 事件发生时间

eventPhase 枚举值：

eventPhase 取值 阶段
0 NONE（没有事件）
1 捕获阶段
2 目标阶段
3 冒泡阶段

鼠标事件对象 MouseEvent

当触发 click / mousemove 事件时，事件对象是：MouseEvent

新增属性 说明
clientX / clientY 视口坐标
pageX / pageY 页面坐标
screenX / screenY 屏幕坐标
button 鼠标按键
ctrlKey 是否按下 Ctrl
shiftKey 是否按下 Shift

button 枚举值：

button 枚举值 含义 鼠标按键
0 主按钮 通常是左键
1 辅助按钮 通常是滚轮中键
2 次按钮 通常是右键
3 浏览器后退键 鼠标侧键
4 浏览器前进键 鼠标侧键

当触发 keydown / keyup 事件时，事件对象是：KeyboardEvent

新增属性 说明
